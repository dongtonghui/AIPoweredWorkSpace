MY WORKFLOW

我的工作流：从 Idea 到 Diff 的 6 步闭环

STEP 01
产品 Idea / 需求目标
STEP 02
与 Hermes 交流 → 输出 PRD
STEP 03
部署到 Git 仓库
STEP 04
输出 Demo → GitHub Pages
STEP 05
继续修改 → 再部署
STEP 06
开发同学看到 Diff
STEP 01 / 07
产品 Idea
需求目标
PM
明确要解决的问题：用自然语言描述需求场景、目标用户和预期效果。不需要先画原型，先想清楚"为什么"。
Output
需求描述 "想做一个会员系统的定价页，支持月付/年付切换"
STEP 02 / 07
与 Hermes 交流
输出 PRD
AI
结合 PRD 相关 skills：把需求抛给 Hermes Agent，通过多轮对话澄清边界、补充场景、细化验收标准，最终输出结构化 PRD。
Output
prd.md https://github.com/dongtonghui/AIPoweredWorkSpace/blob/main/membership-system/prd.md
STEP 03 / 07
部署到
Git 仓库
RD
PRD 成为知识库起点：把 prd.md 提交到 Git 仓库，每次迭代都有完整历史。本质上逐渐形成包含大量产品细节的知识库。
Output
git commit "feat: add membership pricing PRD v1"
STEP 04 / 06
输出 Demo →
GitHub Pages
AI
结合 Demo 设计 skills：基于 PRD 让 Hermes 生成高保真 HTML 原型，可交互、可预览。Demo 文件随仓库推送，GitHub Pages 自动发布为可访问的 HTML 链接。
Output
demo.html https://dongtonghui.github.io/AIPoweredWorkSpace/ring-light/
STEP 05 / 06
继续修改
→ 再部署
PM
持续打磨：看到线上效果后提出调整，Hermes 修改后再 push，GitHub Pages 自动更新。循环直到满意。
Output
git push 每次 push 自动更新线上原型
STEP 06 / 06
开发同学
看到 Diff
RD
改动点一目了然：研发通过 Git diff 精确看到每处变更——改了什么、为什么改、谁改的。无需口头同步。
Output
git diff 行级变更追踪，可评论可评审
← 迭代循环 →
核心收益： PRD + Demo + 迭代历史全部在同一个 Git 仓库里，PM 和研发共享同一套"事实来源"
AI 角色： Hermes 负责生成草稿和原型，PM 负责决策和验收，研发负责评审和实现
研发视角： 不需要等 PM 口头同步，打开 GitHub 就能看到最新原型和全部变更记录
