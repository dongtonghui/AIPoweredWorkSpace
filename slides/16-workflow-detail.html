MY WORKFLOW

我的工作流：从 Idea 到 Diff 的完整闭环

01
产品 Idea
需求目标
PM
明确要解决的问题：用自然语言描述需求场景、目标用户和预期效果。先想清楚"为什么"。
需求描述 "想做一个会员系统的定价页..."
02
与 Hermes 交流
输出 PRD
AI
结合 PRD skills：多轮对话澄清边界、补充场景，输出结构化 PRD。
prd.md 需求背景、功能列表、验收标准
03
部署到
Git 仓库
RD
PRD 成为知识库起点：提交到 Git，每次迭代都有完整历史，逐渐形成产品知识库。
git commit "feat: add pricing PRD v1"
04
输出 Demo
HTML 原型
AI
结合 Demo 设计 skills：基于 PRD 生成高保真 HTML 原型，可交互、可预览。
demo.html 可交互的高保真原型
05
部署到 Git 仓库
→ GitHub Pages
RD
自动部署为在线 URL：推送后 GitHub Pages 自动发布，团队所有人都能访问。
https://... /demo/index.html 在线可访问
06
继续修改
→ 再部署
PM
持续打磨：看到线上效果后提出调整，修改后 push，自动更新。循环直到满意。
git push 每次 push 自动更新线上原型
07
开发同学
看到 Diff
RD
改动点一目了然：通过 Git diff 精确看到每处变更——改了什么、为什么改、谁改的。
git diff 行级变更追踪，可评论可评审
核心收益： PRD + Demo + 迭代历史全部在同一个 Git 仓库里，PM 和研发共享同一套"事实来源"
AI 角色： Hermes 负责生成草稿和原型，PM 负责决策和验收，研发负责评审和实现
研发视角： 不需要等 PM 口头同步，打开 GitHub 就能看到最新原型和全部变更记录
