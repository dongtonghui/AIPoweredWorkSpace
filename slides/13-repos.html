LOCAL ↔ REMOTE IN DETAIL

本地仓库初始化 & 远程同步配置

💻
第一步：初始化本地仓库
在本地创建项目目录并启用 Git 追踪
1
创建项目目录
每个产品/需求独立一个仓库，避免文件混乱。
# 创建并进入项目目录
mkdir acme-product-docs
cd acme-product-docs
2
初始化 Git 仓库
生成 `.git` 目录，开始版本追踪。
git init
Initialized empty Git repository in .../.git/
3
配置用户名和邮箱
让每一次 commit 都带有作者信息。
git config user.name "Kris"
git config user.email "kris@acme.com"
🔐
第二步：配置远程同步凭证
让 Agent 能安全地推送代码到 GitHub
1
在 GitHub 生成 Personal Access Token
路径：GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)。勾选 repo 权限。
2
把 Token 写入 Agent 的环境变量
不要硬编码在代码里，也不要上传到仓库。
# ~/.bashrc 或 ~/.zshrc 或 Agent 的 .env 文件
export GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"
export GITHUB_USER="kris"
安全提醒： Token 相当于密码，务必加入 `.gitignore`，且只给最小必要权限。
3
关联远程仓库
用 HTTPS + Token 的方式关联，Agent 无需交互式输入密码。
git remote add origin https://$GITHUB_USER:$GITHUB_TOKEN@github.com/acme-product/docs.git
🚀
第三步：用自然语言命令 Agent 同步
不用记 Git 命令，直接对 Agent 说话
提交并推送当前改动：
“我已经改好了首页推荐模块的 PRD，请帮我提交到当前分支，commit message 写清楚改动点，然后 push 到远程。”
创建并切换新分支：
“我要开始做一个新需求，叫 checkout-redesign，帮我创建一个分支，并把当前文档复制一份进去。”
拉取最新代码：
“远程 main 分支可能有更新，先帮我 pull 下来，如果有冲突告诉我冲突在哪里。”
发起 PR：
“这个分支的改动已经测试完了，帮我推送到 GitHub 并创建一个 Pull Request，标题写明这是首页推荐模块 PRD，@tech-lead 评审。”
⚙️
Agent 背后的实际执行
自然语言会被翻译为具体 Git 操作
# Agent 解析你的指令后自动执行
git add .
git commit -m "update: 首页推荐模块目标从月活改留存"
git push origin feat/home-recommendation
gh pr create --title "PRD: 首页推荐模块" --reviewer tech-lead
✓
你只需确认，Agent 负责执行
Agent 会先列出将要执行的命令，等你确认后再运行。遇到冲突或权限问题会停下来向你求助。
