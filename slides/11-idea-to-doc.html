IDEA → DOCUMENT

用 Hermes Agent 把一句话需求变成 PRD

01
💬
自然语言描述
PM 用日常语言描述需求，不需要先写模板。AI 负责把模糊想法结构化。
“我想在首页加一个推荐模块，根据用户历史行为推荐商品，提升转化率。”
→
02
🤖
Hermes Agent 生成草稿
Agent 自动补全目标、用户故事、功能范围、验收标准、风险提示，输出 Markdown。
docs/prd/home-recommendation.md
├── 目标与指标
├── 用户故事
├── 功能范围
└── 验收标准
→
03
✏️
PM 审校 + 提交 PR
PM 在 Markdown 里修改关键假设，然后提交 Pull Request，@ 研发负责人评审。
git checkout -b feat/home-rec-prd
git add docs/prd/home-recommendation.md
git commit -m "add: 首页推荐模块 PRD"
→
04
✅
评审通过即定稿
研发在 PR 里提问、PM 回复，全部讨论沉淀在文档行级评论上。merge 后即为团队共识。
✅ Approved by @tech-lead
✅ Merged into main
📌 单源真相已建立
关键提示： AI 生成的是“70 分草稿”，不是终稿。PM 必须审校业务假设、补充上下文，再通过 PR 让团队共同打磨到 90 分。
